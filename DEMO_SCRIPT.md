## 功能演示提纲（按时间顺序追加）

### 1. AI 多轮对话
- **一句话描述**：基于 Kimi K2.6 的流式对话，支持多轮上下文、图片理解、联网搜索。
- **演示要点**：发送消息 → 展示流式输出 → 展示历史上下文连续性。
- **预估耗时**：20 秒

### 2. 结构化数据输出（Generative UI）
- **一句话描述**：通过 JSON Mode 强制 AI 输出结构化数据，前端自动渲染为交互式卡片、雷达图和时间轴。
- **演示要点**：
  1. 切换角色为「简历分析」，输入一段简历文本 → AI 返回 JSON，前端渲染为「简历卡片」（含头像、技能标签、薪资）。
  2. 切换角色为「心理测试」，输入心理状态描述 → AI 返回 JSON，前端渲染为「评分仪表盘 + SVG 雷达图 + 维度进度条」。
  3. 切换角色为「旅游规划」，输入目的地和天数 → AI 返回 JSON，前端渲染为「时间轴行程单」（含时间、景点、预算、贴士）。
- **预估耗时**：60 秒

### 3. Artifacts 独立工作区（Side-by-side Canvas）
- **一句话描述**：当 AI 生成长代码或长文章时，右侧自动滑出独立 Canvas 面板，支持语法高亮、复制、展开/收缩。
- **演示要点**：
  1. 让 AI 写一段超过 20 行的代码（如快速排序）→ 消息hover时出现「Canvas」按钮 → 点击后右侧滑出代码面板。
  2. 展示面板内的语法高亮、一键复制、展开/收缩功能。
  3. 关闭面板后回到正常会话列表，再次打开其他长内容可切换。
- **预估耗时**：30 秒

### 4. 全局个性化记忆（Custom Instructions）
- **一句话描述**：用户可设置「关于我」和「回复偏好」，系统自动拼接到每条请求的 system prompt 中，实现千人千面。
- **演示要点**：
  1. 点击顶部齿轮图标 → 填写「关于我：我是一个 5 岁的小孩」「回复偏好：请用童话故事解释」→ 保存。
  2. 问一个复杂的物理问题（如「什么是黑洞」）→ AI 立刻用童话口吻回答。
  3. 新建会话，再次提问 → 偏好仍然生效，证明是全局记忆而非会话级。
- **预估耗时**：30 秒

### 5. 局部重写 / 文本润色 (Inline Edit)
- **一句话描述**：用户在 AI 回复上划选任意段落，弹出悬浮菜单进行局部修改，模型只重写选中的部分并原地替换。
- **演示要点**：
  1. 让 AI 生成一段文字（如一封邮件草稿）。
  2. 选中其中一段文字 → 弹出「更正式 / 更简短 / 扩写 / 自定义」菜单。
  3. 点击「更正式」→ 等待 1-2 秒 → 选中的文本被替换为更正式的表达，其余内容不变。
  4. 再选中另一段 → 点击「自定义」→ 输入"加入数据支撑" → 文本被替换。
- **预估耗时**：30 秒

### 6. 泛用函数调用框架 (Function Calling)
- **一句话描述**：AI 不再只是"说话的嘴"，而是能主动调用本地 Python 工具完成实际任务（查天气、查库存等）。
- **演示要点**：
  1. 打开左侧「工具 ON」开关。
  2. 输入"北京今天天气怎么样？" → AI 自动调用 `get_weather` 工具 → 返回"晴，25度" → AI 组织成自然语言回复。
  3. 输入"苹果手机还有货吗？多少钱？" → AI 自动调用 `query_inventory` 工具 → 返回库存和价格 → AI 组织成结构化回复。
  4. （可选）强调：比赛日只需在 `agent_tools.py` 里加 Python 函数，前端零改动。
- **预估耗时**：30 秒

### 7. MockMate · 双模式入口（核心差异化）
- **一句话描述**：进入 `/interview` 看到「练习 / 模拟」二选一卡片——产品按用户生命周期切两条主线，命中"高频对练 + 沉浸演练"两个独立场景。
- **演示要点**：
  1. 登录后跳转到 ModeSelect 页，明确指出"这是 ChatGPT 复刻不了的产品形态"——通用 ChatBot 既无法保持面试官人格的跨关一致性，也无法做专项无记忆刻意练习。
  2. 左卡片：练习模式，自由跳转、不带前序记录；右卡片：模拟模式，5 关线性、跨关施压、终局复盘。
- **预估耗时**：15 秒

### 8. 练习模式 · 高频对练（PRACTICE.MODE）
- **一句话描述**：选任意一关直接开练，专攻 STAR 行为面、连刷场景题、找面试感——无前序面试官弱点污染。
- **演示要点**：
  1. 进入 PracticeHub，左侧填「字节跳动 / 前端开发」+ 上传简历 PDF → 一次保存（target 是用户级单例，下次直接复用）。
  2. 点击「情景面」卡片直接进入 → 左侧 input 输「开始」→ AI 现场生成场景题。
  3. 强调：本关 system prompt 没有注入 `prev_reviews`，面试官不会冷不丁说"前面技术官反馈说你抗压差"。
  4. 右上角「重练本关」一键清空、「留档」一键保存到 `practice_logs`。
- **预估耗时**：30 秒

### 9. 模拟模式 · 沉浸演练（MOCK.MODE）
- **一句话描述**：完整 5 关线性面试，后关面试官能看到前关面评/分数/弱点，针对性施压——这是产品的 Wow Moment。
- **演示要点**：
  1. 进入 MockHub → `[ NEW MOCK ]` 创建新场次（公司/岗位/简历）→ 自动跳到第 0 关。
  2. 第 0 关联网生成情报报告，顶部小卡片渲染风格 / 高频考点 / 难度 / 备考建议。
  3. 第 1 关 AI 直读 PDF 提取标签云、风险点、深挖项目。
  4. 第 2 关技术面：故意暴露一个弱点（比如 Redis 持久化答错）→ 右侧雷达图实时画上去 → 「结束本轮」生成结构化面评。
  5. **关键演示**：第 3 关情景面开场，面试官一句"我看到上一关你在 Redis 持久化上答得有些迟疑，那我换个角度"——直接让评审看到跨关记忆在工作。
  6. 强调：顶部 nav 未解锁的关卡是灰色 + 锁图标，**严格阻止跳关**，符合真实面试节奏。
- **预估耗时**：45 秒

### 10. 复盘报告页 · 终局收束
- **一句话描述**：跑完 5 关自动跳转 `/interview/mock/:id/report`，综合雷达 + 5 关面评摘要 + 录用建议大字展示。
- **演示要点**：
  1. 进入复盘报告，顶部右侧大徽章 `[ STRONGLY RECOMMEND / RECOMMEND / ON HOLD / NOT RECOMMEND ]` 一眼定结论。
  2. 综合评分 80 分一个，雷达图覆盖 10 个维度（5 项技术 + 5 项情景）。
  3. KEY.STRENGTHS / KEY.GAPS 两栏对比，FINAL.ADVICE 大段放在底部。
  4. 强调：这是一次完整面试的 ROI——一次跑通拿到一份比 HR 更详细的面评。
- **预估耗时**：20 秒
